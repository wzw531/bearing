## 改动概览

昨天的改动主要集中在两条链路：

1. `ADS124S08` 采集数据打包与 UART DMA 发送链路优化。
2. `IIS3DWB` 振动传感器的 FIFO 批量读取与异常恢复处理。

整体目标看起来是：

- 让振动、应变、温度数据统一进入 `Bearing_Data_Struct` 数据包；
- 避免 CPU 写包和 UART DMA 发包访问同一块内存而导致数据错乱；
- 提升 `IIS3DWB` 高频采样场景下的数据读取稳定性；
- 在 FIFO 溢出或串口发送异常卡住时增加恢复能力。

## 详细改动说明

### 1. `ADS124S08` 数据发送改为双缓冲机制

涉及文件：

- `Core/Inc/ADS124S08.h`
- `Core/Src/ADS124S08.c`
- `Core/Src/main.c`

主要改动：

- 在 `ADS124S08.h` 中新增 `Double_Buffer_Manager` 结构体定义。
- 结构体内部维护两个 `Bearing_Data_Struct` 缓冲区。
- 新增 `active_index`、`ready_index`、`buffer_status[2]`、`pending_transmit` 等状态字段，用于区分“当前采集中”“待发送”“发送中”的缓冲区状态。

改动目的：

- 避免 `Bearing_Data` 在被 UART DMA 发送时，又被采样流程继续写入，导致发出去的数据包不完整或内容交叉。
- 为持续采样场景提供更稳定的异步发送能力。

### 2. 新增 UART DMA 发送状态控制与发送完成回调

涉及文件：

- `Core/Src/ADS124S08.c`

主要改动：

- 新增全局状态：
  - `uart_busy`
  - `uart_tx_start_ms`
- 新增超时参数：
  - `UART_TX_TIMEOUT_MS`，当前值为 `5000 ms`
- 新增 `start_buffer_transmit(uint8_t index)`：
  - 将目标缓冲区状态标记为发送中；
  - 调用 `SCB_CleanDCache_by_Addr(...)` 清理 D-Cache，保证 DMA 读取到的是最新数据；
  - 通过 `HAL_UART_Transmit_DMA(&huart1, ...)` 发包。
- 新增 `HAL_UART_TxCpltCallback(UART_HandleTypeDef *huart)`：
  - 在 DMA 发送完成后，释放对应缓冲区；
  - 自动检查是否还有待发送缓冲区；
  - 如果有待发送数据，则继续启动下一包发送。

改动目的：

- 解决串口 DMA 连续发送场景下的状态切换问题；
- 保证下一包数据可以自动接续发送；
- 补齐 H7 平台上 D-Cache 与 DMA 共用内存时的数据一致性处理。

### 3. 定时器打包发送逻辑增加“忙闲判断”和“超时恢复”

涉及文件：

- `Core/Src/main.c`
- `Core/Src/ADS124S08.c`

主要改动：

- 在 `HAL_TIM_PeriodElapsedCallback()` 的 `TIM6` 分支中：
  - 将当前 `Bearing_Data` 拷贝到待发送缓冲区；
  - 设置缓冲区为 `ready` 状态；
  - 若串口空闲则立即启动 DMA 发送；
  - 若串口忙，则切换到另一个缓冲区等待后续发送。
- 增加 UART 卡死恢复逻辑：
  - 如果 `uart_busy == 1` 且发送持续时间超过 `5000 ms`；
  - 则清理忙状态，并尝试重新发送已就绪缓冲区。

改动目的：

- 减少发送阻塞导致的数据丢包风险；
- 在 DMA 异常未正常回调时，具备一定自恢复能力；
- 提升系统长时间运行时的数据传输稳定性。

### 4. `IIS3DWB` 改为 FIFO 水位触发后的批量读取

涉及文件：

- `Core/Src/iis3dwb.c`
- `Core/Inc/iis3dwb.h`

主要改动：

- 在 `iis3dwb_fifo()` 中配置 FIFO：
  - 设置 FIFO 水位 `FIFO_WATERMARK = 5`
  - 配置批量输出速率 `IIS3DWB_XL_BATCHED_AT_26k7Hz`
  - 配置 FIFO 为 `STREAM_MODE`
  - 配置输出数据率 `IIS3DWB_XL_ODR_26k7Hz`
  - 打开 INT1 的 FIFO 水位中断路由
- 在 `IIS3DWBTR_ReadData()` 中：
  - 读取 FIFO 状态；
  - 根据 `fifo_level` 分批调用 `iis3dwb_fifo_out_multi_raw_get(...)`；
  - 将 FIFO 中的数据写入 `Bearing_Data.adcx[]`；
  - 使用 `IISDWB_Cnt_flag` 管理写入位置，达到 `2132` 后回绕。

改动目的：

- 从“可能逐点读取”切换为“FIFO 批量搬运”，更适合高速振动数据采集；
- 降低中断压力和 SPI 事务次数；
- 让振动数据直接进入统一数据包，便于后续串口发送。

### 5. `IIS3DWB` 增加 FIFO 溢出恢复机制

涉及文件：

- `Core/Src/iis3dwb.c`

主要改动：

- 在 `IIS3DWBTR_ReadData()` 中检查：
  - `fifo_status.fifo_full`
  - `fifo_status.fifo_ovr`
- 如果检测到 FIFO 满或溢出：
  - 先切换到 `IIS3DWB_BYPASS_MODE`
  - 延时 `1 ms`
  - 再切回 `IIS3DWB_STREAM_MODE`
  - 同时将 `IISDWB_Cnt_flag` 清零

改动目的：

- 当采样速度高于处理速度时，避免 FIFO 堆积后持续异常；
- 出现异常后快速恢复采样链路；
- 防止旧数据和新数据混杂。

### 6. 主循环与外设初始化联动调整

涉及文件：

- `Core/Src/main.c`

主要改动：

- 初始化阶段新增：
  - `MX_I2C2_Init();`
  - `IIS3DWB_GPIO_Init();`
  - `iis3dwb_fifo();`
- 主循环中新增 `IIS3DWBTR_INT1flag` 处理：
  - 检测到中断标志后调用 `IIS3DWBTR_ReadData();`
- 外部中断回调 `HAL_GPIO_EXTI_Callback()` 中新增：
  - `IIS3DWBTR_INT1_Pin` 的处理中断标志；
  - `IIS3DWBTR_INT2_Pin` 的状态标记。

改动目的：

- 让 IIS3DWB 从“只初始化”进入“中断驱动 + FIFO 读取”的工作模式；
- 让采集流程在主循环中响应传感器水位中断；
- 为后续扩展 INT2 功能预留入口。
